# E707: Mor terminator presence mismatch

**Last modified:** 2026-05-30 19:04 EDT

## Description

The %mor tier has a terminator but the main tier does not, or vice versa.
One tier ends with a sentence-final punctuation mark while the other does not.

## Metadata

- **Error Code**: E707
- **Category**: Alignment terminator mismatch
- **Level**: tier
- **Layer**: validation
- **Status**: not_implemented

## Example 1

**Trigger**: Main tier has terminator but %mor does not

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	hello .
%mor:	co|hello
@Comment:	ERROR: Main tier has terminator "." but %mor has none
@End
```

## Example 2

**Trigger**: %mor has terminator but main tier does not

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	hello
%mor:	co|hello .
@Comment:	ERROR: %mor has terminator "." but main tier has none
@End
```

## Expected Behavior

These raw CHAT examples no longer reach a distinct E707 diagnostic in the
current parser/model pipeline. `%mor` terminator absence is surfaced earlier as
missing-tier-content / alignment-skipped diagnostics (for example E305/E600),
and when the main tier omits a terminator the `%mor` model synthesizes one
instead of emitting a presence-mismatch error.

**Trigger**: One of main tier or %mor tier has a terminator and the other
does not.

## CHAT Rule

Historically the `%mor` tier mirrored the main tier's terminator presence. In
the current model, `%mor` always carries a terminator and raw-input absence is
handled before alignment. See the CHAT manual:
https://talkbank.org/0info/manuals/CHAT.pdf

## Notes

- Distinct from E716 which checks terminator **value** mismatch (both have
  terminators, but they differ).
- Marked `not_implemented` because the current raw-input pipeline no longer
  emits E707 from CHAT text examples.
