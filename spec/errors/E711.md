# E711: Mor empty content

## Description

A %mor word has an empty stem, POS category, prefix, or suffix. Every
morphosyntax item on the %mor tier must have a non-empty POS category
and a non-empty stem at minimum.

## Metadata
- **Status**: not_implemented
- **Last updated**: 2026-04-13 14:42 EDT
- **Status note**: Unreachable via tree-sitter parser. The grammar rule `mor_lemma` requires non-empty content, so `v|` (empty stem) causes a tier-level parse error (E316) before the model-layer E711 validation runs. Similarly, `|want` (empty POS) is not recognized as a `mor_word` node. The E711 validation exists at the model layer (`model/dependent_tier/mor/tier.rs`) and has unit tests, but the parser's structural validation catches empty POS/lemma before the model is constructed.

- **Error Code**: E711
- **Category**: Mor content validation
- **Level**: tier
- **Layer**: validation

## Example 1

**Trigger**: %mor item with empty stem (POS but no stem after pipe)

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	want cookie .
%mor:	v| n|cookie .
@End
```

## Example 2

**Trigger**: %mor item with empty POS category (no category before pipe)

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	want cookie .
%mor:	|want n|cookie .
@End
```

## Expected Behavior

The parser should successfully parse this CHAT file, but validation should
report an error. Every %mor item must contain both a non-empty POS category
and a non-empty stem.

## CHAT Rule

The %mor tier format is `POS|stem` with optional prefix, suffix, and fusion
markers. Neither POS nor stem may be empty. See the CHAT manual:
https://talkbank.org/0info/manuals/CHAT.pdf

## Notes

- E711 validation logic exists and is tested at the model layer, but the
  tree-sitter grammar's structural requirements prevent empty POS/lemma from
  reaching the model construction phase.
- Related codes: E706 (count mismatch), E708 (malformed grammar relation).
