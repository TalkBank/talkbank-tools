# E711: Mor empty content

## Description

A %mor word has an empty stem, POS category, prefix, or suffix. Every
morphosyntax item on the %mor tier must have a non-empty POS category
and a non-empty stem at minimum.

## Metadata
- **Status**: not_implemented

- **Error Code**: E711
- **Category**: Mor content validation
- **Level**: tier
- **Layer**: validation

## Example 1

**Trigger**: %mor item with empty stem (POS but no stem after pipe)

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	want cookie .
%mor:	v| n|cookie .
@Comment:	ERROR: First %mor item "v|" has an empty stem
@End
```

## Example 2

**Trigger**: %mor item with empty POS category (no category before pipe)

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	want cookie .
%mor:	|want n|cookie .
@Comment:	ERROR: First %mor item "|want" has an empty POS category
@End
```

## Expected Behavior

The parser should successfully parse this CHAT file, but validation should
report an error. Every %mor item must contain both a non-empty POS category
and a non-empty stem.

## CHAT Rule

The %mor tier format is `POS|stem` with optional prefix, suffix, and fusion
markers. Neither POS nor stem may be empty. See the CHAT manual:
https://talkbank.org/0info/manuals/CHAT.pdf

## Notes

- This validates the content of %mor items after successful parsing.
- Related codes: E706 (count mismatch), E708 (malformed grammar relation).
