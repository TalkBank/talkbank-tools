# E716: Mor terminator value mismatch

## Description

The %mor tier has a terminator that does not match the main tier's terminator.
Both tiers have terminators, but they differ (e.g., main tier ends with "?"
but %mor ends with "."). This typically indicates stale or incorrectly cached
morphosyntax data.

## Metadata

- **Error Code**: E716
- **Category**: Alignment terminator mismatch
- **Level**: tier
- **Layer**: validation
- **Status**: not_implemented

## Example 1

**Trigger**: Main tier ends with "?" but %mor ends with "."

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	huh ?
%mor:	co|huh .
@Comment:	ERROR: Main tier terminator is "?" but %mor terminator is "."
@End
```

## Example 2

**Trigger**: Main tier ends with "!" but %mor ends with "."

```chat
@UTF8
@Begin

@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	wow !

```

## Example 3

**Trigger**: Main tier ends with "+..." (trailing off) but %mor ends with "."

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	and then +...
%mor:	conj|and adv|then .
@Comment:	ERROR: Main tier terminator is "+..." but %mor terminator is "."
@End
```

## Expected Behavior

The parser should successfully parse this CHAT file, but validation should
report an error. The %mor tier's terminator must always match the main tier's
terminator exactly.

**Trigger**: Main tier and %mor tier both have terminators, but they differ.

## CHAT Rule

The %mor tier mirrors the main tier's content structure. The terminator on
the %mor line must match the terminator on the main line. See the CHAT manual:
https://talkbank.org/0info/manuals/CHAT.pdf

## Notes

- Distinct from E707 which checks terminator **presence** mismatch (one has
  it, the other doesn't). E716 checks terminator **value** mismatch (both
  have terminators, but they differ).
- This error was discovered via a cache bug where the morphosyntax cache key
  did not include the terminator, causing cached %mor entries from one
  utterance to stamp their terminator onto a different utterance with the
  same words but a different terminator.
