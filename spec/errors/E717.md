# E717: Pho terminator presence mismatch

## Description

The %pho tier has a terminator but the main tier does not, or vice versa.
One tier ends with a sentence-final punctuation mark while the other does not.

## Metadata

- **Error Code**: E717
- **Category**: Alignment terminator mismatch
- **Level**: tier
- **Layer**: validation
- **Status**: not_implemented

## Example 1

**Trigger**: Main tier has terminator but %pho does not

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	hello .
%pho:	hɛloʊ
@Comment:	ERROR: Main tier has terminator "." but %pho has none
@End
```

## Example 2

**Trigger**: %pho has terminator but main tier does not

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|test|CHI||female|||Target_Child|||
*CHI:	hello
%pho:	hɛloʊ .
@Comment:	ERROR: %pho has terminator "." but main tier has none
@End
```

## Expected Behavior

The parser should successfully parse this CHAT file, but validation should
report an error. The %pho tier's terminator presence must match the main
tier's terminator presence.

**Trigger**: One of main tier or %pho tier has a terminator and the other
does not.

## CHAT Rule

The %pho tier mirrors the main tier's content structure. If the main tier
has a terminator, %pho must also have one. See the CHAT manual:
https://talkbank.org/0info/manuals/CHAT.pdf

## Notes

- Parallels E707 (which checks %mor terminator presence mismatch).
- Both E717 and E707 are about presence mismatches, distinct from the value
  mismatch variants (E716 for %mor).
