# E724: GRA has circular dependency

**Last updated:** 2026-04-04 08:28 EDT

## Description

A %gra tier contains a circular dependency where following parent pointers creates a cycle. This violates the fundamental requirement that dependency structures must form a tree.

## Metadata

- **Error Code**: E724
- **Category**: validation
- **Level**: tier
- **Layer**: validation
- **Status**: implemented
- **Severity**: error

## Example 1: Simple 2-Node Cycle

**Source**: Manual example
**Trigger**: Word A points to word B, word B points to word A
**Expected Error Codes**: E724

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|corpus|CHI|||||Target_Child|||
*CHI:	hello there .
%mor:	co|hello adv|there .
%gra:	1|2|DISCOURSE 2|1|ADVMOD 3|1|PUNCT
@End
```

**Analysis:**
Word 1 → 2 (DISCOURSE),
Word 2 → 1 (ADVMOD),
Word 3 → 1 (PUNCT).
Cycle: 1 → 2 → 1.

## Example 2: Complex Multi-Node Cycle

**Source**: Real corpus data (aphasia-data, non-conforming tool output)
**Trigger**: ROOT points to a dependent that points back to ROOT
**Expected Error Codes**: E724

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	PAR Participant
@ID:	eng|corpus|PAR|||||Participant|||
*PAR:	bong , seven .
%mor:	n|bong punct|, num|seven .
%gra:	1|3|FLAT 2|3|PUNCT 3|1|APPOS 4|1|PUNCT
@End
```

**Analysis:**
Word 1 → 3 (FLAT),
Word 2 → 3 (PUNCT),
Word 3 → 1 (APPOS) -- points back to word 1,
Word 4 → 1 (PUNCT).
Cycle: 1 → 3 → 1.

## Example 3: ROOT with Non-Self Head (Python Bug Pattern)

**Source**: Real corpus data (non-conforming tool output)
**Trigger**: Word marked as ROOT but points to another word (not self or 0)
**Expected Error Codes**: E724

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|corpus|CHI|||||Target_Child|||
*CHI:	I want it .
%mor:	pro|I v|want pro|it .
%gra:	1|2|NSUBJ 2|3|ROOT 3|2|OBJ 4|2|PUNCT
@End
```

**Analysis:**
Word 1 → 2 (NSUBJ),
Word 2 → 3 (ROOT) -- ROOT shouldn't point to another word,
Word 3 → 2 (OBJ) -- points back to word 2,
Word 4 → 2 (PUNCT).
Cycle: 2 → 3 → 2.

## Detection Algorithm

To detect circular dependencies:

1. For each non-ROOT word W:
   - Follow parent pointers: W → parent₁ → parent₂ → ...
   - If we ever return to W, there is a cycle
   - If we reach a self-referential ROOT (head=self) or virtual ROOT (head=0), no cycle

2. Valid dependency structures:
   - Must form a tree (no cycles)
   - Must have exactly one root
   - All words must have a path to the root

## Root Cause

This error pattern appears in 591 files across the TalkBank corpus, generated by non-conforming tool output. An off-by-one index mapping error caused UD ROOT (head=0) to wrap around to the last element, creating circular dependencies.

**Impact**: 87.5% of affected morphotag output contains circular dependencies (verified on test files).

## Expected Behavior

**Parser (lenient mode)**: Reports as **E724 warning** (not error). Parser accepts circular dependencies to enable reading and fixing broken corpus files.

**Generator (strict mode)**: ENFORCED with `Result<>`. The Rust generator validates all %gra tiers before returning and returns `Err(MappingError::CircularDependency)` if detected. The caller skips the utterance. This makes it impossible to generate invalid %gra.

## CHAT Rule

Universal Dependencies requires dependency structures to form a tree:
- Each word has exactly one head (parent)
- No cycles allowed
- Exactly one root (word with head=0 or head=self)

See:
- [Universal Dependencies Guidelines](https://universaldependencies.org/)
- [CHAT Manual - Grammatical Relations](https://talkbank.org/0info/manuals/CHAT.html#Grammatical_Relations)
